1
Lab Test Ordered
Doctor prescribes lab tests through the E-Prescription system. The system creates a lab order with status "ordered" and payment status "pending".
POST /api/lab-tests/orders
2
Billing Completed
Lab staff completes the billing process by updating payment status to "paid". This enables the next step in the workflow.
PUT /api/lab-tests/orders/:id/billing
3
Sample Collection Scheduled
After billing is complete, sample collection can be scheduled. The system creates a collection record with default settings.
POST /api/lab-tests/sample-collection
4
Processing & Completion
Lab staff processes the tests and updates status through "in_progress" to "completed". Sample collection status is automatically synchronized.
PUT /api/lab-tests/orders/:id/status

üîß Technical Implementation

Frontend Components
  • LabTestManagement.js - Main component with billing modal and workflow logic
  • LabTestManagement.css - Styling for payment badges and buttons
  • Billing Modal - Complete billing interface with payment methods
  • Smart Buttons - Different actions based on current status
Backend API Endpoints
  • PUT /api/lab-tests/orders/:id/billing - Update payment status
  • POST /api/lab-tests/sample-collection - Create collection record
  • PUT /api/lab-tests/orders/:id/status - Update order status
  • GET /api/lab-tests/orders - Fetch all orders with payment status
Database Schema
  • lab_orders - paymentStatus, totalAmount, status
  • sample_collection - orderId, collectionDate, status
  • lab_order_items - Individual test items in orders
